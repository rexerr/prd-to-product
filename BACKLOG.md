## Format

`BACKLOG.md` is a **thin always-loaded index** ([D-048](docs/DECISIONS.md)). Rules:

- **`## Active` carries one index line per live unit of work** — title · `status` · **Next:** action · pointers. The full context to act lives in a `tickets/<slug>.md` or the cited `D-NNN`/retro, never inline here.
- **Thin-on-touch (binding):** editing a `## Legacy` fat entry converts it to an index line (+ ticket if it needs working context) as part of that edit. Leaving it fat after touching it is a rule violation, not a deferral.
- **Retire, don't re-accrete:** resolved work moves to a retro or `tickets/archive/` (archive, don't delete) via the `/end-session` retirement ritual. Done items do not return to this file.
- **Status field** carries the old In-progress-vs-Backlog distinction: `in-progress` (actively being worked) · `next` (promotion criterion met, queued) · `watching` (promotion trigger named, waiting on it).
