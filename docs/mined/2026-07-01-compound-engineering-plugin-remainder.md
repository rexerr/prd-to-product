### R-17 — Agent-friendly CLI principles · soft (synthesized rubric, sourced) · → build-defaults row

`docs/solutions/agent-friendly-cli-principles.md` is a self-contained **7-principle rubric for CLIs that AI agents consume**, synthesized from Anthropic's tool-design guidance + the CLI Guidelines project (clig.dev): (1) non-interactive by default for automation paths (`--no-input`, detect non-TTY and never prompt, `--yes`/`--force`); (2) structured parseable output (`--json`, exit 0/non-0, result→stdout / diagnostics→stderr, suppress color when not a TTY); (3) progressive help discovery (layered `--help`, at least one concrete example per non-trivial subcommand); (4) fail-fast **actionable** errors (state what was wrong + the correct invocation + valid values); (5) safe retries + explicit mutation boundaries (`--dry-run`, idempotent create/update, audit-friendly IDs on append/trigger); (6) composable/predictable structure (stdin, `-` alias, consistent verb/resource naming); (7) bounded high-signal responses (limits/filters/pagination; a default over ~500 lines is a friction signal unless the command is bulk-oriented). It carries a citable stance: **CLI beats MCP for most dev-facing agent work** — an MCP server can burn tens of thousands of tokens loading tool definitions before the first question, while a CLI call costs only the command + its output; MCP earns its cost only when per-user auth / structured governance is needed. Each principle ships a Blocker/Friction/Optimization severity ladder and a runnable check. **Soft-source** (a rubric, not a verified repo behavior) — lands as build-defaults guidance for a scaffolded project that ships a CLI, and as a checklist for our own scripts.

The other three root files were low-yield through our lens: `codex-skill-prompt-entrypoints.md` (Codex native-install specifics — dedup with the cross-host converter already covered); `plugin-versioning-requirements.md` (release-please discipline — we don't distribute via versioned releases); `adding-converter-target-providers.md` (converter internals — dedup; minor nuggets: "structural mapping over heuristic section-extraction," and they *do* `forceSymlink` for personal skill sync — a small counterpoint to ASCE's blanket anti-symlink stance).
