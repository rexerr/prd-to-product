## The headline — the decision-grade finding (feeds Seq-1 project-setup)

gstack is a **near-complete, heavily-dogfooded, MIT, markdown-only answer to the exact project-setup gap** the trio leaves open — and it settles two things the earlier [foundation council](../council/council-report-2026-06-30-foundation.html) got wrong.

**It closes 4 of the trio's 5 holes with substantive (not gestural) commands:**

- **Security** — `/cso` runs a 15-phase (0–14) OWASP Top-10 + STRIDE + supply-chain + LLM-security checklist with **per-phase false-positive suppression rules** and precedent references (`cso/sections/audit-phases.md:40,65,80,94,110`). Analysis-only, no code changes. Plus a 7-category `/review` security specialist + an adversarial red-team pass.
- **Audit** — `/review` dispatches **parallel fresh-context specialist subagents** with confidence scoring (1–10, suppress <5), **adaptive gating** (a 0-finding specialist auto-gates, but security + data-migration are `[NEVER_GATE]` insurance), and ingests external Greptile/codex findings (`review/SKILL.md`, `review/checklist.md`, `review/specialists/`).
- **Test/QA** — `/qa` drives a real headless/CDP browser against live + staging URLs (`$B goto`/`snapshot`/`fill`/`console --errors`), scores health, keeps JSON regression baselines, and **bootstraps a test framework + CI + `TESTING.md` from scratch when the project has none** (`qa/SKILL.md:934-1087`).
- **Deploy** — `/setup-deploy`→`/ship`→`/land-and-deploy`→`/canary` is a genuine config-to-verified-in-prod chain: detects fly/render/vercel/etc., waits on CI (`gh pr checks --watch`), canary-verifies the live site, and has a real `git revert` rollback path (`land-and-deploy/SKILL.md:886+`).
- Plus `/guard`+`/careful`+`/freeze` — enforced by **real `PreToolUse` blocking hooks** (`careful/bin/check-careful.sh` returns `{"permissionDecision":"ask"}` on `rm -rf`/`DROP TABLE`/`git push --force`), not prose.

**It shares the trio's 5th hole:** *nobody scaffolds the stack.* gstack explicitly refuses — `office-hours/SKILL.md:876,1690`: *"Do NOT... scaffold any project... Not even scaffolding."* Both gstack and the trio stop at "make the *plan* rigorous, let the model free-hand the skeleton." **Stack-init is the one gap neither fills** — go look for it elsewhere (a TanStack Start/Router *app* starter, per the [template mine](2026-07-01-tanstack-template.md)).

**Its maintenance plumbing is the existence-proof the council doubted:** a solo non-engineer *can* keep a downstream fleet current — `VERSION` file (`1.58.5.0`) checked against a remote channel + install-type-aware `/gstack-upgrade` + a fail-safe SessionStart auto-update (`bin/gstack-session-update`: forks to background, `THROTTLE_SECONDS=3600`, **`exit 0` always** so a network failure never blocks a session) + versioned idempotent migrations. Team mode **de-vendors** any committed copy (`bin/gstack-team-init` → `git rm --cached` + `.gitignore`), so upgrades can't clobber user work because gstack never sits in the user's tree.

**The compose-vs-build fork sharpens to a third option.** Adopting gstack *itself* is all-or-nothing — one clone drags in a mandatory Bun toolchain, a compiled browser stack (Playwright + Puppeteer + a 22MB→721MB ML prompt-injection classifier), 74 bash bins, opt-in-but-present telemetry to a Garry-Tan-hosted Supabase, gbrain, and a **router preamble that actively tells the agent to prefer gstack skills over answering** (`SKILL.md.tmpl:38`) — which would fight our own CLAUDE.md session discipline. **So: compose the *methodology* as a reference architecture; don't vendor the *artifact* as a dependency.** The maintenance model is ~200 lines of portable bash; the capability commands are *blueprints*, not lift-and-shift (their `~/.gstack/` machinery would breach the D-013/D-019 don't-import-internal-vocabulary line).

**This is a genuine fork (costly + hard to reverse) → recommend a council at the actual decision point** (per the "recommend a council at genuine forks" rule). Not now — the mine fed it.

## Findings

### G-1 — Deploy Configuration block in the scaffolded CLAUDE.md · code-grounded (Lens B) · ADOPTED (→ row)

**Source:** `setup-deploy/SKILL.md:922-937` — detects the platform and persists a typed `## Deploy Configuration` block (platform / prod URL / deploy-trigger / health-check / merge-method) into CLAUDE.md that later skills *read* instead of re-detecting. Platform-agnostic: reads config, hardcodes no stack.

**Why it matters:** our scaffold produces context + rules but no deploy facts. A detected-and-confirmed deploy block is a clean, transferable addition that begins to fill the deploy hole — and it models the broader **platform-agnostic contract** (read config from CLAUDE.md → AskUserQuestion if missing → persist back) that is the "compose capabilities per project type" shape implemented as a *convention*, not a code layer.

**Landed as:** board row — *Deploy Configuration block in scaffolded CLAUDE.md* (`backlog`, `area:generator`).

### G-2 — Skill-authoring upgrades (bucket) · mixed · ADOPTED (→ watching row, gate:rule-of-2)

Four skill-craft borrows, single-source so parked under rule-of-two; fold in on the next skill/template edit:

- **Richer rule grammar (`code-grounded`)** — gstack pairs each security/review check not just with a failure mode but with **what NOT to flag** (precedent-backed FP-suppression clauses, `cso/sections/audit-phases.md:40`) **and the CI tripwire that enforces it** (`CLAUDE.md` rules cite the exact test that fails if the rule is violated). This is a direct extension of our "every rule cites its failure mode" invariant — a rule that says what not to flag is what keeps a generated `.claude/rules/` from crying wolf.
- **Preamble-injection by tier (`code-grounded`)** — every skill declares `preamble-tier: 1–4` in frontmatter (`SKILL.md.tmpl:3`); a generator (`scripts/resolvers/preamble.ts`) composes shared discipline (ethos, voice, AskUserQuestion format) whose depth scales with the tier. A *mechanical* fix for the copy-paste drift our "port self-improvements back to the skill" rule fights by hand.
- **AskUserQuestion decision-card grammar (`code-grounded`)** — every user decision renders as a fixed card: ELI10 · stakes-if-wrong · lettered recommendation · "options differ in kind, not coverage" note (`skillify/SKILL.md.tmpl:87-102`). Independently echoes Rex's own prefs (recommend a path; frame decisions in terms I own).
- **context-save / context-restore (`code-grounded`)** — forward-looking WIP handoff tooling (append-only `checkpoints/<ts>-<slug>.md`; save defaults to current-branch, restore to all-branches; ordered by filename timestamp not mtime). Complements our backward-looking retros; both could coexist in a scaffold.

**Landed as:** board row — *Skill-authoring upgrades from gstack mine* (`watching`, `gate:rule-of-2`, `area:skill-templates`).

### G-3 — Real `PreToolUse` blocking-hook model · code-grounded (Lens A/B) · ADOPTED (→ folded)

**Source:** `careful/bin/check-careful.sh` — a self-contained bash hook that reads tool JSON from stdin and returns `{"permissionDecision":"ask"}` to block destructive commands (`rm -rf`, `DROP TABLE`, `git push --force`, `git reset --hard`, `kubectl delete`), with a build-artifact allowlist exception.

**Landed as:** a note under the existing board row *On-demand hook scaffolds in the skill* (no new row).

### Guard — do NOT import gstack's completeness ethos

gstack's ETHOS is **"Boil the Ocean"** (completeness-is-cheap, `ETHOS.md:34-60`) with **no line/file scope gate** and an explicit **rejection of a hard file-size cap** (soft 160KB/~40K-token *warning* on generated skills, justified by prompt-cache economics — `CLAUDE.md:161`). This is a genuine philosophical fork *opposite* our ≤3-file/≤50-line + ≤500-line discipline, tuned for compiled cache-hit prompts, not hand-maintained docs. **Do not import Boil-the-Ocean framing into a scaffold that ships our scope-gate discipline.**

### Dedup / dropped (no action)

- **`/review` vs our `/code-review`; `/cso` vs `/security-review`; `/qa` vs `verify`** — real overlaps; the *delta* worth noting is architectural (parallel fresh-context specialists + adaptive gating + external-tool ingestion; infrastructure-first checklist exhaustiveness; health-scoring + regression baselines + framework bootstrap). Not re-proposed — we ship the built-ins.
- **`/retro`** — a team/velocity dashboard, a different animal from our failure-tag session retros. No steal.
- **Delivery model** — gstack clones the whole repo into `~/.claude/skills/gstack` and generates per-skill dirs with symlinked `SKILL.md`; our per-skill symlink model already gets atomic-per-skill updates, but we have no VERSION/upgrade channel on top. The maintenance model (existence-proof above) is the transferable asset, captured in the headline as Seq-1 design input (connects to the *Brownfield context-drift* row).
- **gstack-specific plumbing (DO NOT transfer):** `~/.gstack/` telemetry/analytics/gbrain/decision-log, the compiled `$B`/`$D` binaries, Bun toolchain, host-specific Conductor/OpenClaw branches, the router preamble. gstack's own product surface, not portable capability.
