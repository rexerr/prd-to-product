# Running design-research engagements as Claude Code projects — brief

## Purpose

Run client engagements that are design, research, synthesis, and prototyping work as Claude Code projects, so context compounds across sessions instead of restarting from memory each time. Each engagement is scaffolded bespoke from a shared starter; the reusable pattern gets harvested out of real runs, not designed up front. Three kinds of artifact, kept separate: research (rigor, cited), stimulus (disposable prototypes), and handoff (the precise, versioned deliverable).

## The skills we want

Engagement skills, project-local, invoked by hand:

- `/ingest-call` — turn a stakeholder call into decisions, owners, and backlog updates.
- `/ingest-session` — turn a research session into a de-identified record that keeps observation separate from interpretation.
- `/synthesize` — cluster sessions into cited, confidence-tagged themes, contradictions kept visible.
- `/prototype-brief` — turn a concept into a disposable test stimulus tied to the hypothesis it tests.
- `/standup` and `/postcard` — a morning briefing and a client status note, added later when the need is felt.

Personal skills, user-level so they travel to every project, invoked by hand:

- `/capture` — dump a stray thought to an inbox before it derails or vanishes.
- `/next` — surface exactly one next action, nothing else.
- `/smallest-step` — break an avoided task into an absurdly small first move.
- `/focus` — body-double, work with me on one thing for 25 minutes.
- `/welcome-back` — shame-free re-entry after a gap, here's the one thing.

Most skills are manual-invoke, so they cost nothing in context until called. Build `/capture` and `/next` first; they attack the bottleneck and prove the habit fastest.

## Background, why it's built this way

The system is shaped for an ADHD-pattern way of working, where the hard part is almost never the task itself, it's the cost of starting. A board of twelve cards raises that cost; it doesn't lower it. So the design choices all reduce activation energy and protect against things falling out of sight:

- Kill the blank page. Every skill drafts first and lets you react. Editing is far cheaper to start than creating, and that's the single biggest thing AI does for this kind of brain.
- One next action, not a board. The default surface is a single line for the frozen moment, not a wall to scan.
- Push, not pull. Briefings come to you, because anything off-screen stops existing.
- Make time visible. Time blindness cuts both ways, so show elapsed and remaining on dependencies and deadlines.
- Re-entry over guilt. After a gap the system says "welcome back, here's the one thing," never a scold. Shame just becomes another reason to avoid the work.

These keep the system from being the thing you procrastinate on. The same instinct guards the build itself: ship thin versions, add a skill only after you've felt the friction it removes, and let the engagement, not the tooling, set the pace.
