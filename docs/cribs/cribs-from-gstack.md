## The meta-theme

gstack converged on one idea our verification doctrine only states from one side:

> **A claim — success OR finding — is worth nothing until its evidence is named.** Our CLAUDE.md already forbids the *success* claim ("never claim success because the code looks correct"). gstack's review/cso/plan-review skills supply the symmetric, *finding-side* discipline: every assertion must quote the verbatim source line that motivates it, or be force-demoted and suppressed; "looks fine" is as banned as "looks done." Independent convergence (the same concern raised by ≥2 deliberately-blinded reviewers) is treated as the strongest signal the system produces — a rule this very mine demonstrated, since seven independent readers surfaced the same handful of mechanisms.

**The dominant tension, stated once:** gstack's central ethos — **"Boil the Ocean / completeness is cheap / the complete thing is the goal"** — is the structural *opposite* of this workspace's hard scope discipline (bug ≤3 files/≤50 lines; feature ≤300 lines). Every gstack *scope* mechanism (auto-approve in-blast-radius expansions, "ship the complete version") is an **anti-crib** here. The cribbable value is almost entirely in *review-pass plumbing* and *interview discipline*, which are scope-neutral and survive our invariants intact. The divergences are recorded at the bottom, not adopted.

## Confirmed / already-held (one line each, citing the existing crib — do NOT mint a new ID)

- **Calibration-event loop** (`review`/`cso`) — a finding reported at confidence <7 the user confirms is real is a logged "calibration event" → fold into **C-22** (provenance/confidence vocab) as the *feedback rule that corrects the vocab over time*. (Convergent across 2 readers.)
- **"Handles ≠ is the deliverable" + path-concreteness** (`review` Verification Mode) — code that *handles* a deliverable isn't the deliverable; a named concrete path must be DONE/NOT-DONE via `[ -f path ]`, "I didn't check" ≠ "unreachable" → sharpens **C-04**.
- **2-consecutive-confirmation before flagging** (`canary`) — alert only on a pattern that persists across ≥2 checks (transient suppression) → refines **C-08** (golden should-not-change baseline).
- **Verify the *landed* artifact + no-silent-rollback** (`skillify` Step 11) — folded into G-20 above; the no-rollback clause also extends **C-18 / CF-13**.
- **Greenfield escape-hatch for explore-before-ask** (`spec` Phase 3) — "I searched X/Y/Z, found nothing, treating as greenfield" prevents explore-before-ask stalling on a no-code project → delta on **CF-18**.
- **Reference-grade bar** (`document-generate`) — "accepts a string (max 256, `^[a-z-]+$`)" not "accepts a string"; no *why* in reference docs → content discipline *inside* a **CF-29** FORMAT file, not a new crib.
- **Reference-docs-first / read-the-tests-as-behavior-spec** (`document-generate`) — minor deltas on **CF-18** / explore-before-ask.
- **Forced minimal-vs-ideal option spread** (`openclaw-office-hours` Phase 4) — "one option must be the smallest-diff version" → refinement to our existing "always recommend a path."
- **User-Challenge inversion** (`autoplan` 6 principles) — when both models agree the user's stated direction should change, that's NEVER auto-decided; the human's direction is the default and the models must make the case (what-you-said / both-recommend / why / what-we-might-miss / cost-if-wrong) → sharpens **C-16** (decision-ready handoff). This is gstack's ETHOS "User Sovereignty"; we already hold the spirit ("always gated on Rex").
- **Convergence-guard for review loops** (`plan-ceo-review`) — same issues returned twice → STOP, persist as named "Reviewer Concerns", present anyway → extends CLAUDE.md loop-detection.
- **"Most recent = filename timestamp, not mtime"** (`context-restore`) — thin durable-correctness note for retro/handoff file selection (stable across copy/rsync). Confirms the recent same-name-overwrite-forbidden retro convention from the other side.
